#​‌‍​‌‌​‍‌‍‍​​‍‌‍‌‍‌‍‌​‌‌​​‍‌‌‍‍‌​​​‍‍​‌‍ Bowling Alley Challenge

A 4-app hexagonal system demonstrating clean architecture with zero cross-app imports.

## Architecture
Built using Dr. Alistair Cockburn's Hexagonal Architecture (Ports & Adapters).

- **Inside**: Pure domain logic and typed ports.
- **Outside**: Adapters (In-memory storage, Express HTTP API, CLI).
- **Communication**: Bridge adapters in the Front Desk hexagon connect to other apps.
- **Composition Root**: `main.ts` wires everything together.

## Apps
1. **Player Registry**: Manages bowlers and shoe sizes.
2. **Lane Manager**: Tracks lane status (available, in-use, maintenance).
3. **Scoring Engine**: Implements real bowling scoring (strikes, spares, 10th frame).
4. **Front Desk**: Orchestrates the game flow.

## Public API
- `POST /games`: Book a new game.
- `POST /games/:id/rolls`: Record a roll.
- `GET /games/:id/scoreboard`: View the current score.
